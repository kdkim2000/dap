# 6과목 4장: 데이터 품질 관리 관점

## 1. 표준 관점 품질관리 프로세스

**표준 관점**의 품질관리는 데이터가 조직에서 정의한 표준(단어·용어·코드·도메인)을 준수하는지 관리하는 것이다.

### 1.1 표준 관점 품질 측정 항목

| 측정 항목 | 설명 |
|------|------|
| 표준 용어 준수율 | 표준 용어로 정의된 속성 비율 |
| 표준 코드 준수율 | 표준 코드 목록 내 값 비율 |
| 표준 도메인 준수율 | 표준 도메인으로 정의된 컬럼 비율 |
| 명명 규칙 준수율 | 테이블·컬럼명이 명명 규칙을 준수한 비율 |

### 1.2 표준 관점 품질관리 절차

```
1. 표준 정의 (표준 단어·용어·코드·도메인)
2. 현행 데이터 구조와 표준 비교 (GAP 분석)
3. 미준수 항목 식별 및 개선 계획 수립
4. 표준 적용 및 데이터 정제
5. 준수율 측정 및 모니터링
```

### 1.3 데이터 표준화 원칙

| 원칙 | 설명 |
|------|------|
| 유일성 | 표준 단어·용어는 유일한 의미를 가져야 함 |
| 완전성 | 모든 업무 데이터에 표준이 적용되어야 함 |
| 일관성 | 동일한 의미의 데이터는 동일한 표준을 사용해야 함 |
| 명확성 | 표준 정의는 모호하지 않아야 함 |

## 2. 모델 관점 품질관리 프로세스

**모델 관점**의 품질관리는 데이터 모델(개념·논리·물리)이 업무 요구사항을 정확하게 반영하고 품질 기준을 충족하는지 관리하는 것이다.

### 2.1 모델 관점 품질 기준

| 품질 기준 | 설명 |
|------|------|
| 완전성 | 모든 업무 요구사항이 모델에 반영되어 있는가 |
| 정확성 | 엔터티·속성·관계 정의가 업무를 정확히 표현하는가 |
| 일관성 | 개념·논리·물리 모델 간 불일치가 없는가 |
| 표준 준수 | 표준 용어·도메인·명명 규칙을 준수하는가 |
| 추적성 | 요구사항과 모델 요소가 연결되어 있는가 |

### 2.2 모델 품질 검토 항목

**논리 데이터 모델 검토**:
- 식별자 누락 여부
- 불필요한 중복 속성
- 정규화 위반 여부 (1NF, 2NF, 3NF)
- 관계 누락·오류
- 서브타입 정의의 적절성

**물리 데이터 모델 검토**:
- 테이블·컬럼명 표준 준수
- 데이터 타입·길이 적절성
- 인덱스 누락 여부
- 논리 모델과의 일치성

### 2.3 모델 품질관리 절차

```
1. 모델 품질 기준 정의
2. 모델 검토 (동료 검토, 전문가 검토)
3. 품질 이슈 식별 및 분류
4. 이슈 해결 (모델 수정)
5. 수정 결과 확인
6. 품질 수준 측정·기록
```

## 3. 값 관점 품질관리 프로세스

**값 관점**의 품질관리는 실제 저장된 데이터 값의 정확성·완전성·유효성 등을 관리하는 것이다.

### 3.1 값 관점 품질 측정 지표

| 지표 | 측정 방법 |
|------|------|
| 정확성 | 실제 값과 기준 값 불일치 비율 |
| 완전성 | NULL 또는 공백 값 비율 |
| 유효성 | 허용 범위·형식을 벗어난 값 비율 |
| 일관성 | 시스템 간 동일 데이터의 불일치 비율 |
| 유일성 | 중복 레코드 비율 |

### 3.2 데이터 프로파일링(Data Profiling)

데이터의 구조·내용·품질을 분석하는 기법.

| 분석 유형 | 내용 |
|------|------|
| 컬럼 분석 | 최솟값·최댓값·평균·NULL 비율·고유값 수 |
| 패턴 분석 | 데이터 형식 패턴 분석 |
| 규칙 분석 | 업무 규칙 준수 여부 확인 |
| 관계 분석 | 테이블 간 참조 무결성 확인 |

### 3.3 값 관점 품질관리 절차

```
1. 품질 기준 정의 (측정 지표, 허용 임계치)
2. 데이터 프로파일링 (현황 파악)
3. 품질 이슈 식별 (기준 미달 데이터)
4. 원인 분석 (입력 오류, 시스템 오류 등)
5. 데이터 정제 (수정, 삭제, 보완)
6. 예방 대책 수립 (입력 검증 강화 등)
7. 지속 모니터링
```

## 4. 활용 관점 품질관리 프로세스

**활용 관점**의 품질관리는 데이터가 업무 활동과 의사결정에 적절하게 활용되고 있는지 관리하는 것이다.

### 4.1 활용 관점 품질 측정 항목

| 측정 항목 | 설명 |
|------|------|
| 접근성 | 필요한 데이터에 적시에 접근 가능한가 |
| 이해성 | 데이터의 의미를 사용자가 이해할 수 있는가 |
| 신뢰성 | 사용자가 데이터를 신뢰하고 활용하는가 |
| 적시성 | 데이터가 필요한 시점에 최신 상태인가 |
| 유용성 | 데이터가 업무 목적에 부합하는가 |

### 4.2 활용 관점 품질 향상 방안

| 문제 | 해결 방안 |
|------|------|
| 접근 어려움 | 데이터 카탈로그 구축, 셀프서비스 BI 도입 |
| 의미 불명확 | 비즈니스 메타데이터 정비, 데이터 사전 제공 |
| 낮은 신뢰도 | 데이터 품질 측정 결과 공개, 품질 개선 이력 관리 |
| 최신성 부족 | 배치 주기 단축, 실시간 연계 전환 |

### 4.3 데이터 카탈로그(Data Catalog)

조직의 데이터 자산을 목록화하고 검색할 수 있게 하는 시스템.
- 데이터 위치, 의미, 품질 수준 정보 제공
- 데이터 활용 촉진

---

## 5. 4가지 관점 종합 비교

| 관점 | 주요 관리 대상 | 핵심 측정 항목 |
|------|------|------|
| 표준 관점 | 표준 단어·용어·코드·도메인 준수 | 표준 준수율 |
| 모델 관점 | 데이터 모델 품질 | 완전성·정확성·일관성 |
| 값 관점 | 실제 저장된 데이터 값 | 정확성·완전성·유효성 |
| 활용 관점 | 데이터 활용 적절성 | 접근성·신뢰성·유용성 |

---

## 출제 포인트

1. **4가지 품질관리 관점**: 표준·모델·값·활용
2. **표준 관점**: 표준 준수율 측정 (용어·코드·도메인·명명규칙)
3. **모델 관점 품질 5기준**: 완전성·정확성·일관성·표준 준수·추적성
4. **값 관점**: 데이터 프로파일링, 정확성·완전성·유효성 측정
5. **활용 관점**: 접근성·이해성·신뢰성·적시성·유용성
6. **데이터 프로파일링**: 컬럼·패턴·규칙·관계 분석
7. **데이터 카탈로그**: 데이터 자산 목록화, 검색·활용 촉진
8. **표준화 원칙 4가지**: 유일성·완전성·일관성·명확성
9. **값 관점 품질 향상**: 입력 검증 강화, 데이터 정제
10. **모델 품질관리 절차**: 기준정의→검토→이슈식별→해결→측정
