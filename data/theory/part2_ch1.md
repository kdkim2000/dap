# 2과목 1장: 정보 요구 사항 개요

## 1. 정보 요구 사항 정의

**정보 요구 사항(Information Requirement)** 은 조직의 업무 목표를 달성하기 위해 필요한 정보의 내용, 구조, 품질, 타이밍에 관한 요구를 체계적으로 정의한 것이다.

### 1.1 정보 요구 사항의 개념

| 구분 | 설명 |
|------|------|
| 정의 | 업무 수행에 필요한 정보의 요구를 체계적으로 기술한 것 |
| 목적 | 데이터 아키텍처·시스템 설계의 기반 마련 |
| 범위 | 데이터 항목, 관계, 규칙, 품질, 보안 등 |

### 1.2 정보 요구 사항의 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 기능적 정보 요구 | 업무 기능 수행에 필요한 정보 | 주문 처리에 필요한 고객·상품 정보 |
| 비기능적 정보 요구 | 정보 품질·성능·보안에 관한 요구 | 응답시간 3초, 데이터 암호화 |
| 데이터 표준 요구 | 데이터 명칭·형식·코드 표준화 요구 | 표준 날짜 형식 YYYY-MM-DD |
| 데이터 품질 요구 | 정확성·완전성·일관성 확보 요구 | 주요 필드 NULL 허용 불가 |

### 1.3 정보 요구 사항과 다른 요건의 관계

| 요건 유형 | 관계 |
|---------|------|
| 기능 요건(FR) | 정보 요구 사항과 긴밀히 연계 — 기능 수행에 필요한 정보 도출 |
| 비기능 요건(NFR) | 정보의 품질·성능·보안 요구 포함 |
| 업무 규칙 | 정보 생성·변경·삭제 조건 정의 → 데이터 요건 도출 근거 |

## 2. 정보 요구 사항 관리

정보 요구 사항 관리는 수집된 요구 사항을 체계적으로 저장·추적·변경·검증하는 활동이다.

### 2.1 정보 요구 사항 관리의 필요성

| 이유 | 내용 |
|------|------|
| 일관성 유지 | 여러 시스템 간 정보 요구의 일관성 확보 |
| 변경 통제 | 요구 사항 변경 시 영향 분석·승인 체계 |
| 추적성 확보 | 요구 사항 → 설계 → 구현 연결 관계 관리 |
| 품질 보증 | 도출된 정보 요구 사항의 정확성·완전성 검증 |

### 2.2 정보 요구 사항 관리 항목

| 관리 항목 | 설명 |
|---------|------|
| 요구 사항 ID | 고유 식별자 |
| 요구 사항 명칭 | 간결한 명칭 |
| 요구 사항 설명 | 상세 내용 |
| 우선순위 | 중요도 (High/Medium/Low) |
| 출처 | 요구 사항 제시자·근거 문서 |
| 상태 | 검토 중/승인/구현/완료 |
| 검증 기준 | 충족 여부 확인 방법 |

### 2.3 정보 요구 사항 추적성(Traceability)

요구 사항이 설계·구현·테스트 등 개발 산출물과 어떻게 연결되는지 관리하는 것.

```
정보 요구 사항 → 데이터 모델(ERD) → 물리 테이블 → 테스트 케이스
```

| 추적 방향 | 목적 |
|---------|------|
| 순방향(Forward) | 요구 사항이 올바르게 구현되었는지 확인 |
| 역방향(Backward) | 설계·구현 요소의 근거 요구 사항 파악 |

### 2.4 정보 요구 사항 우선순위 결정 기법

**MoSCoW 기법:**

| 구분 | 의미 |
|------|------|
| Must Have | 반드시 구현해야 하는 필수 요구 사항 |
| Should Have | 중요하지만 없어도 운영 가능 |
| Could Have | 있으면 좋은 요구 사항 |
| Won't Have | 이번 범위에서 제외 (미래 고려) |

### 2.5 정보 요구 사항의 품질 기준

| 기준 | 설명 |
|------|------|
| 명확성(Clarity) | 모호하지 않고 단일한 해석 가능 |
| 완전성(Completeness) | 필요한 모든 정보 포함 |
| 일관성(Consistency) | 다른 요구 사항과 충돌 없음 |
| 검증가능성(Verifiability) | 충족 여부를 테스트로 확인 가능 |
| 추적가능성(Traceability) | 출처와 구현 연결 추적 가능 |

## 출제 포인트

1. 정보 요구 사항 유형: 기능적·비기능적·데이터 표준·데이터 품질
2. 정보 요구 사항 관리 항목: ID·명칭·설명·우선순위·출처·상태·검증기준
3. 추적성(Traceability): 요구사항↔설계↔구현↔테스트 연결 관계
4. 순방향 추적: 요구사항이 올바르게 구현되었는지 확인
5. 역방향 추적: 설계·구현 요소의 근거 요구 사항 파악
6. MoSCoW: Must/Should/Could/Won't Have 4단계 우선순위
7. 정보 요구 품질 기준: 명확성·완전성·일관성·검증가능성·추적가능성
8. 업무 규칙이 데이터 요건 도출 근거
9. 정보 요구 관리 필요성: 일관성·변경통제·추적성·품질보증
10. 요구 사항 상태 관리: 검토 중→승인→구현→완료
