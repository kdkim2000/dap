# 4과목 1장: 데이터 모델링 이해

## 1. 데이터 모델링 개요

### 1.1 데이터 모델링 정의

**데이터 모델링(Data Modeling)** 은 현실 세계의 데이터를 추상화하여 컴퓨터에서 처리 가능한 형태로 표현하는 과정이다.

| 구분 | 설명 |
|------|------|
| 목적 | 업무에서 필요한 데이터를 체계적으로 정의 |
| 결과 | 데이터 모델(ERD 등) 산출 |
| 활용 | 데이터베이스 설계의 기반 |

### 1.2 데이터 모델링의 3가지 특징

| 특징 | 설명 |
|------|------|
| 추상화(Abstraction) | 현실 세계를 일정한 형식으로 표현 |
| 단순화(Simplification) | 약속된 표기법으로 간결하게 표현 |
| 명확화(Clarity) | 애매모호함을 제거하고 명확히 기술 |

### 1.3 데이터 모델링 3단계

```
1. 개념 데이터 모델링  →  2. 논리 데이터 모델링  →  3. 물리 데이터 모델링
(주제 영역·핵심 엔터티)   (엔터티·속성·관계 상세화)   (테이블·컬럼·인덱스 설계)
```

| 단계 | 목적 | 산출물 | 특징 |
|------|------|--------|------|
| 개념 | 핵심 데이터 개념 정의 | 개념 ERD | DBMS 독립적 |
| 논리 | 업무 규칙·속성 상세화 | 논리 ERD | DBMS 독립적 |
| 물리 | DB 구현을 위한 설계 | 테이블 명세서 | DBMS 종속적 |

## 2. 데이터 모델링 기법 이해

### 2.1 엔터티-관계(ER) 모델링

**ERD(Entity-Relationship Diagram)** 은 엔터티와 이들 간의 관계를 도식화한 모델링 기법이다.

**핵심 구성 요소:**

| 요소 | 정의 | ERD 표현 |
|------|------|---------|
| 엔터티(Entity) | 업무에서 관리해야 할 독립적 관심 대상 | 사각형 |
| 속성(Attribute) | 엔터티의 특성을 나타내는 데이터 항목 | 타원형 (원래 표기법) |
| 관계(Relationship) | 엔터티 간의 연관 | 마름모 (원래 표기법) |

**엔터티 유형:**

| 유형 | 설명 | 예시 |
|------|------|------|
| 유형(Tangible) 엔터티 | 물리적 형태가 있는 대상 | 고객, 직원, 상품 |
| 개념(Conceptual) 엔터티 | 추상적 개념 대상 | 보험 상품, 서비스 |
| 사건(Event) 엔터티 | 업무 행위·사건으로 생성 | 주문, 거래, 계약 |

**식별자(Identifier) 유형:**

| 유형 | 설명 |
|------|------|
| 본질 식별자 | 업무 속성으로 자연스럽게 결정 |
| 인조 식별자 | 인위적으로 부여 (시퀀스, UUID) |
| 단일 식별자 | 하나의 속성으로 구성 |
| 복합 식별자 | 두 개 이상 속성의 조합 |

**식별자 특성:**

| 특성 | 의미 |
|------|------|
| 유일성(Uniqueness) | 각 인스턴스를 유일하게 식별 |
| 최소성(Minimality) | 식별에 필요한 최소 속성으로 구성 |
| 불변성(Immutability) | 값이 변경되지 않아야 함 |
| 존재성(Not Null) | 반드시 값이 존재해야 함 |

### 2.2 정규화(Normalization)

데이터 중복과 이상 현상(Anomaly)을 제거하기 위한 테이블 분해 과정.

**이상 현상 유형:**

| 이상 유형 | 설명 |
|---------|------|
| 삽입 이상(Insertion Anomaly) | 불필요한 데이터 없이 삽입 불가 |
| 수정 이상(Update Anomaly) | 중복 데이터 일부만 수정 시 불일치 |
| 삭제 이상(Deletion Anomaly) | 하나의 데이터 삭제 시 다른 데이터도 삭제 |

**정규형 단계:**

| 정규형 | 조건 |
|--------|------|
| 제1정규형(1NF) | 모든 속성이 원자값(Atomic Value) |
| 제2정규형(2NF) | 1NF + 부분 함수 종속 제거 |
| 제3정규형(3NF) | 2NF + 이행 함수 종속 제거 |
| BCNF | 3NF + 모든 결정자가 후보 키 |

## 3. 데이터 모델링 표기법 이해

### 3.1 IE 표기법 (Information Engineering)

현재 가장 널리 사용되는 ERD 표기법. 관계선 끝에 기호로 카디널리티를 표현한다.

**카디널리티 기호:**

| 기호 | 의미 |
|------|------|
| `│` (수직선) | 1 (반드시 하나) |
| `O` (원) | 0 (없을 수도 있음, Optional) |
| `<` (까마귀 발) | 多 (하나 이상) |

**관계 표현 예시:**

| 관계 | 기호 조합 |
|------|---------|
| 1:1 (필수) | `│──│` |
| 1:M (선택, 필수) | `O──<` |
| 1:M (필수, 선택) | `│──O<` |
| M:N | `>O──O<` |

**참여도(Participation):**

| 구분 | 의미 |
|------|------|
| 필수 참여 | 반드시 관계에 참여해야 함 (NOT NULL FK) |
| 선택 참여 | 관계에 참여하지 않을 수도 있음 (NULL 허용 FK) |

### 3.2 Barker 표기법

Richard Barker가 제안한 표기법. Oracle에서 많이 사용된다.

**주요 특징:**

| 요소 | 표현 |
|------|------|
| 엔터티 | 모서리가 둥근 사각형 |
| 속성 | 엔터티 내부에 나열, 필수 속성 `*`, 선택 속성 `o` |
| 식별자 | `#` 기호 |
| 관계 | 실선(필수), 점선(선택) |

### 3.3 IE 표기법 vs Barker 표기법 비교

| 구분 | IE 표기법 | Barker 표기법 |
|------|---------|------------|
| 엔터티 모양 | 직사각형 | 둥근 직사각형 |
| 관계 카디널리티 | 까마귀 발 기호 | 실선·점선 |
| 사용 환경 | 범용 | Oracle 중심 |
| 속성 표현 | 별도 나열 | 박스 내 표현 |

### 3.4 식별 관계 vs 비식별 관계

| 구분 | 설명 | IE 표기 |
|------|------|---------|
| 식별 관계 | 부모 PK가 자식 PK의 일부 구성 | 실선 |
| 비식별 관계 | 부모 PK가 자식의 일반 속성(FK) | 점선 |

## 4. 관계형 데이터 모델 심화

### 4.1 릴레이션(Relation)의 특성 6가지

**열(Column)의 특징:**
1. 각 열은 하나의 값만 보유 (원자성)
2. 동일한 종류의 값만 포함 (동질성)
3. 유일한 이름 보유
4. 열의 순서는 무의미

**행(Row)의 특징:**
5. 테이블 내 각 행은 유일해야 함 (중복 행 없음)
6. 행의 순서는 무의미

### 4.2 관계 연산자(Relational Algebra)

수학의 집합 이론을 배경으로 데이터를 집합 단위로 처리:

| 연산자 | 설명 |
|--------|------|
| Select(σ) | 조건에 맞는 행 선택 |
| Project(π) | 특정 열 선택 |
| Product(×) | 두 릴레이션의 카티션 곱 |
| Join(⋈) | 공통 속성으로 두 릴레이션 결합 |
| Union(∪) | 두 릴레이션의 합집합 |
| Intersection(∩) | 두 릴레이션의 교집합 |
| Difference(−) | 두 릴레이션의 차집합 |
| Division(÷) | 나눗셈 연산 |

**처리 연산자:** Insert / Update / Delete

### 4.3 관계형 모델의 무결성 규칙

| 무결성 종류 | 설명 |
|---------|------|
| **엔터티 무결성** | 기본키(PK)는 NULL 값을 포함할 수 없음 |
| **참조 무결성** | 외래키(FK) 값은 반드시 참조 테이블의 PK 값으로 존재해야 함 (NULL 허용) |
| **도메인 무결성** | 컬럼 값은 정의된 데이터 타입·길이·허용값 제약을 준수 |
| **연쇄 작용** | 입력·삭제·수정·조회 작업 시 관련 테이블에 미치는 영향 규칙 |

## 출제 포인트

1. 데이터 모델링 3특징: 추상화·단순화·명확화
2. 데이터 모델링 3단계: 개념→논리→물리 (DBMS 독립→종속)
3. 엔터티 3유형: 유형(물리적)·개념(추상)·사건(행위)
4. 식별자 4특성: 유일성·최소성·불변성·존재성
5. 삽입/수정/삭제 이상: 정규화로 제거
6. 1NF: 원자값, 2NF: 부분 함수 종속 제거, 3NF: 이행 함수 종속 제거
7. BCNF: 모든 결정자가 후보 키
8. IE 표기법: 수직선(1), 원(0), 까마귀 발(多)
9. 필수 참여: NOT NULL FK, 선택 참여: NULL 허용 FK
10. 식별 관계: 부모 PK → 자식 PK 일부 구성 (실선)
11. 비식별 관계: 부모 PK → 자식 일반 속성 FK (점선)
12. Barker 표기법: 둥근 사각형, `#`(식별자), `*`(필수), `o`(선택)
