# 3과목 3장: 데이터 표준 관리

## 1. 데이터 표준 관리 체계

데이터 표준 관리 체계란 수립된 데이터 표준이 지속적으로 유지·발전될 수 있도록 하는 조직, 프로세스, 도구의 종합 체계이다.

> **핵심**: 데이터 표준은 한 번 수립하면 끝나는 것이 아니라, 비즈니스 변화에 맞게 지속적으로 관리·진화해야 한다.

### 데이터 표준 관리의 주요 영역

| 관리 영역 | 내용 |
|--------|-----|
| 표준 유지 관리 | 표준 사전의 지속적 갱신 |
| 표준 준수 관리 | 시스템이 표준을 준수하는지 모니터링 |
| 표준 변경 관리 | 표준 변경 절차 및 영향 관리 |
| 표준 배포 관리 | 표준 공표·교육·공유 |

### 데이터 표준 관리 시스템(메타데이터 관리 시스템)

- **목적**: 데이터 표준 사전을 시스템으로 관리하여 접근성·일관성 향상
- **주요 기능**:
  1. 표준 사전 등록·조회·수정
  2. 버전 관리 및 변경 이력
  3. 표준 준수 자동 검사
  4. 이해관계자 접근 통제

## 2. 데이터 품질 관리

### 데이터 품질의 개념

데이터 품질(Data Quality)은 데이터가 정의된 요건을 충족하는 정도로, 데이터 관리의 핵심 성과 영역이다.

### 데이터 품질 측정 항목

#### 완전성(Completeness)

- **정의**: 필수 데이터 항목이 빠짐없이 존재하는 정도
- **측정 공식**: 완전성 = (값이 있는 건수 / 전체 건수) × 100
- **예시**: 고객 테이블에서 이메일 항목의 채움률

#### 정확성(Accuracy)

- **정의**: 데이터 값이 실제 세계의 참 값과 일치하는 정도
- **측정 방법**: 표준 원천과 대조, 현장 확인
- **예시**: 고객 주소가 실제 주소와 일치하는 비율

#### 일관성(Consistency)

- **정의**: 동일한 데이터가 여러 시스템에서 동일한 값을 가지는 정도
- **측정 방법**: 시스템 간 데이터 비교
- **예시**: ERP와 CRM 시스템의 고객 정보 일치율

#### 유효성(Validity)

- **정의**: 데이터 값이 정의된 도메인·형식·규칙을 준수하는 정도
- **측정 방법**: 도메인 규칙 위반 건수 집계
- **예시**: 날짜 형식 위반, 코드값 범위 이탈

#### 유일성(Uniqueness)

- **정의**: 데이터 개체가 중복 없이 단 한 번만 존재하는 정도
- **측정 방법**: 기본키 중복 검사, 비즈니스 키 중복 검사
- **예시**: 동일 고객이 두 개 이상의 레코드로 존재하는 비율

#### 적시성(Timeliness)

- **정의**: 데이터가 필요한 시점에 최신 상태를 반영하는 정도
- **측정 방법**: 데이터 지연 시간 측정
- **예시**: 주문 상태 변경 후 화면 반영까지 소요 시간

```
데이터 품질 지표 산출 예시:

완전성:  (1000 - 50) / 1000 × 100 = 95%
정확성:  (1000 - 8) / 1000 × 100 = 99.2%
일관성:  (500 - 12) / 500 × 100 = 97.6%
유효성:  (1000 - 30) / 1000 × 100 = 97%
유일성:  (1000 - 5) / 1000 × 100 = 99.5%
```

### 데이터 품질 관리 프레임워크

```
계획(Plan)
  - 품질 목표 설정
  - 측정 항목 정의
  - 측정 주기 결정
    ↓
측정(Do)
  - 데이터 프로파일링
  - 품질 지표 산출
  - 문제 데이터 식별
    ↓
분석(Check)
  - 품질 목표와 비교
  - 근본 원인 분석
  - 개선 우선순위 결정
    ↓
개선(Act)
  - 데이터 정제(Data Cleansing)
  - 프로세스 개선
  - 입력 통제 강화
```

## 3. 데이터 표준 준수 관리

### 준수 관리의 개념

데이터 표준 준수 관리는 시스템 개발 및 운영 단계에서 데이터 표준이 실제로 적용되는지를 검토하고 위반 사항을 개선하는 활동이다.

### 준수 검사 시점

| 검사 시점 | 내용 |
|--------|-----|
| 설계 단계 | ERD, 테이블 명세서의 표준 준수 검토 |
| 개발 단계 | 개발된 DB 스키마와 표준 자동 비교 |
| 운영 단계 | 주기적 표준 준수율 측정 및 보고 |

### 준수 검사 항목

- **명칭 준수**: 테이블명, 컬럼명이 표준 명칭을 사용하는가?
- **도메인 준수**: 컬럼의 데이터 유형·길이가 도메인 표준과 일치하는가?
- **코드 준수**: 코드값이 표준 코드 사전에 정의된 값을 사용하는가?
- **무결성 준수**: 필수 항목에 NOT NULL, FK 제약이 설정되어 있는가?

### 표준 준수율 측정

```
표준 준수율 = (표준 준수 항목 수 / 전체 항목 수) × 100

예시:
- 전체 컬럼 수: 500개
- 표준 준수 컬럼 수: 470개
- 표준 준수율: 470 / 500 × 100 = 94%
```

## 4. 데이터 표준 변경 관리

### 변경 관리의 필요성

데이터 표준은 비즈니스 환경 변화, 법규 개정, 새로운 서비스 출시 등으로 변경이 발생한다. 변경 관리가 미흡하면 시스템 오류, 데이터 불일치가 발생할 수 있다.

### 변경 관리 프로세스

```
1. 변경 요청
   - 변경 요청자: 업무 담당자, 개발자, DA
   - 요청 내용: 변경 이유, 변경 내용, 긴급도
     ↓
2. 영향 분석
   - 변경 시 영향받는 시스템·테이블·프로그램 목록 작성
   - 영향 범위가 클수록 신중한 검토 필요
     ↓
3. 검토 및 승인
   - 데이터 스튜어드 검토
   - 관련 부서 협의
   - 데이터 관리자(DA) 최종 승인
     ↓
4. 변경 적용
   - 표준 사전 갱신
   - 관련 시스템 반영 (배포 계획 수립)
     ↓
5. 변경 통보
   - 관련 이해관계자에게 변경 사항 공지
   - 변경 이력 기록
```

### 변경 유형별 관리

| 변경 유형 | 예시 | 관리 포인트 |
|--------|-----|----------|
| 용어 추가 | 새 비즈니스 용어 등록 | 중복 여부 검토 |
| 용어 수정 | 의미 변경, 영문명 변경 | 기존 사용처 영향 분석 |
| 용어 폐기 | 더 이상 사용하지 않는 용어 | 물리 삭제 대신 폐기 이력 관리 |
| 코드 추가 | 새 코드값 등록 | 기존 범위와 충돌 여부 |
| 도메인 변경 | 데이터 유형·길이 변경 | DB 스키마 변경 영향 분석 |

## 5. 데이터 거버넌스

### 데이터 거버넌스의 정의

데이터 거버넌스(Data Governance)란 조직 전체의 데이터 자산에 대한 의사결정 권한, 책임, 프로세스를 정의하고 실행하는 체계이다.

> **핵심**: 데이터 거버넌스는 "누가 데이터에 대해 책임을 지고, 어떤 규칙으로 데이터를 관리하는가"를 정의하는 종합 체계이다.

### 데이터 오너십(Data Ownership)

- **정의**: 특정 데이터 자산에 대해 비즈니스 책임을 지는 역할
- **데이터 오너의 책임**:
  1. 데이터의 비즈니스 정의 승인
  2. 데이터 접근 정책 결정
  3. 데이터 품질 목표 설정
  4. 데이터 표준 변경 승인

```
데이터 오너십 체계:

데이터 오너 (임원/부서장 수준)
    │ 비즈니스 책임
    ↓
데이터 스튜어드 (실무 담당자 수준)
    │ 일상적 데이터 관리
    ↓
데이터 쿠스토디언 (IT 담당자 수준)
    │ 물리적 데이터 관리
    ↓
데이터베이스 관리자(DBA)
    물리적 저장·보안·백업
```

### 데이터 스튜어드십(Data Stewardship)

- **정의**: 데이터 오너를 대신하여 일상적인 데이터 관리를 수행하는 역할
- **데이터 스튜어드의 책임**:
  1. 데이터 품질 모니터링
  2. 표준 준수 현황 파악
  3. 데이터 이슈 해결
  4. 메타데이터 갱신

| 역할 | 수준 | 주요 책임 |
|-----|-----|---------|
| 데이터 오너 | 비즈니스 (임원) | 데이터 정책 결정, 최종 책임 |
| 데이터 스튜어드 | 비즈니스 (실무자) | 일상적 품질·표준 관리 |
| 데이터 쿠스토디언 | IT (DBA/개발자) | 물리적 저장·보안·백업 |

### 데이터 거버넌스 프레임워크

```
비전 및 전략
  └→ 데이터 원칙 정의
       └→ 데이터 정책 수립
            └→ 표준 및 절차 정의
                 └→ 실행 및 모니터링
```

### 데이터 거버넌스 핵심 구성 요소

| 구성 요소 | 설명 |
|--------|-----|
| 조직(People) | 데이터 위원회, 오너, 스튜어드, 쿠스토디언 |
| 프로세스(Process) | 표준 관리, 품질 관리, 변경 관리 절차 |
| 기술(Technology) | 메타데이터 관리 시스템, 품질 측정 도구 |
| 정책(Policy) | 데이터 사용 정책, 보안 정책, 보관 정책 |

## 6. 출제 포인트

1. **데이터 품질 6대 측정 항목**: 완전성, 정확성, 일관성, 유효성, 유일성, 적시성 — 각 정의와 측정 방법
2. **데이터 오너십**: 비즈니스 책임, 임원/부서장 수준, 데이터 접근 정책 결정 권한
3. **데이터 스튜어드십**: 오너 대신 일상적 데이터 관리, 실무 담당자 수준
4. **데이터 쿠스토디언**: IT 수준, 물리적 저장·보안·백업 담당
5. **표준 변경 관리 5단계**: 요청 → 영향 분석 → 검토/승인 → 적용 → 통보
6. **준수 관리 3가지 시점**: 설계 단계, 개발 단계, 운영 단계
7. **데이터 거버넌스 구성 요소**: 조직(People), 프로세스, 기술, 정책의 4요소
8. **품질 관리 PDCA**: Plan(목표 설정) → Do(측정) → Check(분석) → Act(개선)
9. **코드 폐기 원칙**: 물리 삭제 금지, 사용여부 플래그로 폐기 관리
10. **완전성 공식**: (값이 있는 건수 / 전체 건수) × 100으로 수치 계산 문제 출제
