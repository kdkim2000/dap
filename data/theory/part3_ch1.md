# 3과목 1장: 데이터 표준화 개요

## 1. 데이터 표준화의 정의와 목적

데이터 표준화(Data Standardization)란 조직 내에서 사용되는 데이터의 명칭, 형식, 의미, 규칙을 일관되게 정의하고 관리하는 활동이다.

> **핵심**: 데이터 표준화는 "같은 데이터를 같은 이름으로, 같은 형식으로, 같은 의미로" 사용하도록 조직 전체에 통일성을 부여하는 활동이다.

### 데이터 표준화의 정의

| 기관 | 정의 |
|-----|------|
| ISO | 데이터 요소, 데이터 형식, 데이터 교환에 관한 규칙의 체계적 정립 |
| 국내 행안부 | 기관에서 생성·수집·활용하는 데이터의 명칭, 정의, 형식, 규칙에 대한 원칙과 기준 수립 |

### 데이터 표준화의 목적

1. **데이터 공유 및 연계 촉진**: 기관·부서 간 데이터 교환 시 해석 혼란 방지
2. **데이터 품질 향상**: 일관된 입력 규칙으로 오류 데이터 감소
3. **개발 생산성 향상**: 표준화된 데이터 구조로 신규 시스템 개발 효율화
4. **유지보수 비용 절감**: 일관된 명칭으로 이해도 향상 → 수정 비용 절감
5. **데이터 거버넌스 기반 확립**: 데이터 관리 정책의 기술적 기반

## 2. 데이터 표준화의 필요성

### 데이터 표준화 부재 시 문제점

1. **데이터 사일로(Data Silo)**: 부서별로 독립적인 데이터 관리, 연계 불가
2. **동음이의어 문제**: 같은 용어가 부서마다 다른 의미로 사용
   - 예: "고객" = 영업부서(잠재고객 포함) vs 관리부서(계약 체결 고객만)
3. **이음동의어 문제**: 같은 데이터를 다른 명칭으로 사용
   - 예: 고객ID = 거래처번호 = 클라이언트코드
4. **형식 불일치**: 동일한 데이터가 시스템마다 다른 형식
   - 예: 날짜 형식 (YYYYMMDD vs YYYY-MM-DD vs MM/DD/YYYY)
5. **의미 불명확**: 컬럼명만으로 의미를 파악하기 어려움
   - 예: AMT, AMT1, AMT2, AMT_X

> **핵심**: 데이터 표준화의 핵심 필요성은 동음이의어와 이음동의어 문제 해소이다.

### 데이터 표준화 부재의 비용

- **연계 개발 비용 증가**: 시스템 간 데이터 변환 매핑 작업
- **데이터 정제 비용**: 표준화 이전 데이터 정화 작업
- **의사결정 오류 위험**: 잘못 해석된 데이터 기반 의사결정

## 3. 데이터 표준화 대상

### 데이터 표준(Data Standard)

- **정의**: 데이터 명칭, 형식, 규칙에 대한 원칙과 기준
- **구성 요소**:
  - 용어 표준
  - 단어 표준
  - 도메인 표준
  - 코드 표준

### 데이터 사전(Data Dictionary)

- **정의**: 조직 내 모든 데이터 항목에 대한 메타데이터 집합
- **포함 내용**:
  - 항목명 (한글명, 영문명)
  - 데이터 유형 및 길이
  - 허용값 및 제약 조건
  - 비즈니스 정의
  - 소관 부서 및 담당자
  - 관련 항목

```
데이터 사전 예시:

항목명: 고객번호
영문명: CUSTOMER_NO
데이터유형: VARCHAR(20)
허용값: 영문+숫자 조합, 'CUS' 접두어
필수여부: 필수
비즈니스 정의: 고객을 식별하는 유일한 번호로, 최초 가입 시 자동 부여
소관부서: 고객서비스팀
관련항목: 고객명, 가입일자
```

### 코드 체계(Code System)

- **정의**: 특정 도메인의 값을 표준화된 코드로 표현하는 체계
- **필요성**:
  - 입력 오류 방지
  - 저장 공간 효율화
  - 데이터 통계 처리 용이
- **예시**:

| 코드명 | 코드값 | 의미 |
|------|------|-----|
| 성별코드 | M | 남성 |
| 성별코드 | F | 여성 |
| 주문상태코드 | ORD | 주문 접수 |
| 주문상태코드 | PAY | 결제 완료 |
| 주문상태코드 | DLV | 배송 중 |
| 주문상태코드 | COM | 배송 완료 |

| 표준화 대상 | 설명 | 산출물 |
|---------|-----|-------|
| 데이터 표준 | 명칭·형식·규칙 원칙 | 데이터 표준 지침서 |
| 데이터 사전 | 항목별 메타데이터 | 데이터 사전 |
| 코드 체계 | 도메인 값 표준화 | 코드 정의서 |

## 4. 데이터 표준화 원칙

### 기본 원칙

1. **유일성**: 동일 개념은 하나의 표준 용어로만 표현
2. **일관성**: 조직 전체에 동일한 표준 적용
3. **완전성**: 모든 데이터 항목이 표준에 정의
4. **명확성**: 용어의 의미가 명확하고 모호하지 않음
5. **안정성**: 빈번한 표준 변경 지양, 변경 시 절차 준수

### 데이터 표준화 명명 규칙

- **한글명**: 비즈니스 용어, 공백 없이 명사형
- **영문명**: 대문자 또는 소문자 일관성, 언더스코어 구분
- **약어 사용**: 허용 약어 목록 관리, 임의 약어 사용 금지

```
명명 규칙 예시:
- 테이블명: 명사형, 스네이크 케이스 (CUSTOMER_ORDER)
- 컬럼명: 단어+단어+... 조합 (CUST_ID, ORD_DT)
- 인덱스명: IX_테이블명_컬럼명 (IX_CUSTOMER_CUST_ID)
- FK명: FK_자식테이블명_부모테이블명
```

## 5. 데이터 표준화 조직

### 데이터 표준화 조직 체계

```
데이터 표준화 위원회 (최고 의사결정)
    │
데이터 관리자(DA: Data Administrator)
    │
데이터 스튜어드(Data Steward)
    │
데이터 표준 담당자 (실무)
```

### 주요 역할

| 역할 | 책임 |
|-----|------|
| 데이터 표준화 위원회 | 표준 정책 승인, 예외 사항 결정 |
| 데이터 관리자(DA) | 표준 지침 수립, 전사 표준 관리 |
| 데이터 스튜어드 | 업무 영역별 표준 적용·감독 |
| 데이터베이스 관리자(DBA) | 물리 구현 시 표준 준수 |
| 개발자 | 프로그램에서 표준 명칭 사용 |

> **핵심**: 데이터 관리자(DA)는 전사 수준의 데이터 표준 정책을 수립하고, 데이터 스튜어드는 업무 영역별로 표준을 관리하는 실무자이다.

## 6. 데이터 표준화 추진 단계

데이터 표준화는 계획 수립부터 유지보수까지 체계적 단계로 추진된다.

### 표준화 추진 5단계

```
1단계: 현황 분석
  - 기존 데이터 항목 수집·분석
  - 표준화 위반 현황 파악
  - 동음이의어·이음동의어 식별
    ↓
2단계: 표준화 원칙 수립
  - 명명 규칙 정의
  - 도메인 표준 정의
  - 표준화 지침서 작성
    ↓
3단계: 표준 데이터 정의
  - 표준 용어 사전 구축
  - 코드 체계 설계
  - 데이터 사전 작성
    ↓
4단계: 표준 적용
  - 신규 시스템에 표준 적용
  - 기존 시스템 표준화 전환
  - 표준화 교육 실시
    ↓
5단계: 표준 유지보수
  - 표준 준수 모니터링
  - 표준 변경 관리
  - 지속적 개선
```

### 현황 분석 산출물

- 데이터 항목 현황표
- 동음이의어/이음동의어 목록
- 비표준 항목 현황 (약어, 불명확 명칭)
- 표준화 우선순위 목록

## 7. 데이터 표준 관리 도구(표준화 도구)

### 데이터 표준화 실천 방법

1. **기본 방침 설정**: 데이터 표준화·규격화를 위한 전사 기본 방침 수립
2. **전사적 등록·관리 체계 구축**: 표준 단어·용어·도메인·코드를 등록·관리하는 체계 구축
3. **승인된 데이터 요소 활용**: 승인된 표준 데이터 요소만 사용하도록 강제

### 데이터 표준 4가지 원칙 (명칭/정의/형식/규칙)

| 원칙 | 주요 내용 |
|------|---------|
| **데이터 명칭** | 해당 개념을 유일하게 구분, 보편적으로 인지되는 이름, 명칭만으로 의미와 범위 파악 가능 |
| **데이터 정의** | 제3자 입장에서 기술, 의미 정의와 실제 값을 함께 기술, 약어·전문용어 최소화 |
| **데이터 형식** | 타입·길이·소수점 자리 규정, 유사 데이터 형식 통일, 특수 데이터 타입(CLOB 등) 사용 최소화 |
| **데이터 규칙** | 기본값(Default) / 허용값(Allowed Values) / 허용범위(Range) 정의 |

### 데이터 표준화 구성요소 3가지

| 구성요소 | 설명 |
|---------|------|
| **데이터 표준** | 용어·단어·도메인·코드 표준화 |
| **표준화 절차** | 요구사항 수집 → 표준 정의 → 표준 확정 → 표준 관리 |
| **표준 관리 조직** | 위원회·데이터관리자·데이터스튜어드·DBA 등 역할 조직 |

### 데이터 관리자(Data Administrator) 역할 5가지

1. 표준화 원칙 및 표준 정의
2. 변경·추가 시 최종 승인 의사결정
3. 표준 준수 여부 지속적 관리
4. 데이터 보안 정책 수립 및 관리
5. 데이터 활용 문제 검토 및 방안 수립

## 8. 출제 포인트

1. **데이터 표준화의 3대 대상**: 데이터 표준, 데이터 사전, 코드 체계
2. **동음이의어 vs 이음동의어**: 동음이의어(같은 이름, 다른 의미) vs 이음동의어(다른 이름, 같은 의미)
3. **데이터 사전**: 항목명, 데이터유형, 허용값, 비즈니스 정의, 소관부서 등 메타데이터 집합
4. **코드 체계의 필요성**: 입력 오류 방지, 저장 효율화, 통계 처리 용이
5. **데이터 관리자(DA)**: 전사 수준 데이터 표준 정책 수립·관리
6. **데이터 스튜어드**: 업무 영역별 데이터 표준 적용·감독 실무자
7. **표준화 5단계**: 현황 분석 → 원칙 수립 → 표준 정의 → 적용 → 유지보수
8. **데이터 표준 4원칙**: 명칭·정의·형식·규칙 (무엇을 표준화할 것인가)
9. **표준화 원칙**: 유일성, 일관성, 완전성, 명확성, 안정성
9. **표준화 목적**: 데이터 공유·연계 촉진, 품질 향상, 개발 생산성 향상
10. **표준화 부재 문제**: 데이터 사일로, 연계 개발 비용 증가, 의사결정 오류 위험
