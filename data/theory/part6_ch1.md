# 6과목 1장: 데이터 이해

## 1. 데이터 품질 관리 프레임워크

**데이터 품질 관리(Data Quality Management)** 는 데이터의 정확성·완전성·일관성·적시성을 체계적으로 관리하는 활동이다.

### 1.1 데이터 품질 프레임워크 구성

| 구성 요소 | 설명 |
|------|------|
| 데이터 품질 목표 | 조직이 달성해야 할 데이터 품질 기준 정의 |
| 데이터 품질 측정 | 품질 지표(KPI) 설정 및 측정 방법 결정 |
| 데이터 품질 진단 | 현재 품질 수준 평가 및 문제점 식별 |
| 데이터 품질 개선 | 오류 수정, 표준화, 정제 활동 |
| 데이터 품질 모니터링 | 지속적 품질 수준 추적 및 관리 |

### 1.2 데이터 품질 기준 차원

| 품질 차원 | 설명 | 측정 예시 |
|------|------|------|
| 정확성(Accuracy) | 데이터가 실제 사실을 정확히 반영 | 오류 데이터 비율 |
| 완전성(Completeness) | 필수 값이 누락 없이 존재 | NULL 비율, 공백 비율 |
| 일관성(Consistency) | 여러 시스템 간 데이터가 동일 | 시스템 간 불일치 건수 |
| 적시성(Timeliness) | 데이터가 필요한 시점에 최신 상태 | 데이터 지연 시간 |
| 유효성(Validity) | 데이터가 정의된 형식·범위에 부합 | 형식 오류 비율 |
| 유일성(Uniqueness) | 중복 데이터 없이 유일 | 중복 레코드 비율 |

> **핵심**: 데이터 품질의 6가지 차원 — 정확성, 완전성, 일관성, 적시성, 유효성, 유일성

## 2. 표준 데이터

**표준 데이터(Standard Data)** 는 조직 내 데이터 표준화를 위해 정의된 데이터로, 데이터 표준화 정책과 규칙에 따라 관리된다.

### 2.1 데이터 표준화 구성 요소

| 구성 요소 | 설명 | 예시 |
|------|------|------|
| 표준 단어 | 업무에서 사용하는 단어 목록과 약어 표준 | 고객→CUST, 번호→NO |
| 표준 용어 | 표준 단어 조합으로 만든 업무 용어 | 고객번호=CUST_NO |
| 표준 코드 | 코드 유형과 코드 값의 표준 정의 | 성별코드: M/F |
| 표준 도메인 | 데이터 유형·길이·형식 표준 | 날짜: DATE, 금액: NUMBER(15,2) |

### 2.2 표준 단어 정의 시 고려사항

1. 동의어 처리: 같은 의미의 여러 단어 중 하나만 표준으로 채택
2. 약어 규칙: 공식적인 약어 체계 수립
3. 외래어 처리: 한글/영어 표기 기준 통일
4. 금지어 설정: 표준화에 부적합한 단어 지정

## 3. 모델 데이터

**모델 데이터(Model Data)** 는 데이터 모델(개념·논리·물리)에 정의된 엔터티, 속성, 관계 등의 메타 정보이다.

### 3.1 모델 데이터의 유형

| 유형 | 설명 |
|------|------|
| 개념 데이터 모델 정보 | 주제 영역, 핵심 엔터티, 관계 |
| 논리 데이터 모델 정보 | 속성, 식별자, 관계, 정규화 규칙 |
| 물리 데이터 모델 정보 | 테이블, 컬럼, 인덱스, 파티션 |

### 3.2 모델 데이터 품질 기준

- **완전성**: 모든 엔터티·속성·관계가 정의되어 있는가
- **일관성**: 논리 모델과 물리 모델이 일치하는가
- **표준 준수**: 표준 용어·도메인을 준수하는가
- **추적성**: 요구사항과 데이터 모델이 연결되어 있는가

## 4. 관리 데이터

**관리 데이터(Management Data)** 는 데이터 관리를 위한 메타데이터로, 데이터의 출처·이력·책임자 등을 포함한다.

### 4.1 메타데이터 유형 및 예시

| 유형 | 설명 | 구체적 예시 |
|------|------|------|
| 기술 메타데이터 | 데이터 구조, 형식, 크기 정보 | 테이블명·컬럼 정의·데이터 타입·인덱스 정보·파티션 구조 |
| 업무 메타데이터 | 데이터의 업무적 의미, 정의, 소유자 | 데이터 사전·비즈니스 용어 정의·데이터 소유자·업무 규칙 |
| 운영 메타데이터 | 생성 일시, 수정 이력, 접근 이력 | 생성일시·최종수정자·접근 로그·배치 실행 이력·처리 건수 |
| 관리 메타데이터 | 데이터 책임자, 보관 기간, 보안 등급 | 보안 등급(공개/내부/기밀)·보존 기간·개인정보 해당 여부·암호화 여부 |

### 4.2 데이터 거버넌스 역할

| 역할 | 책임 |
|------|------|
| 데이터 오너(Data Owner) | 데이터에 대한 최종 책임 보유 |
| 데이터 스튜어드(Data Steward) | 데이터 품질·표준 관리 실무 담당 |
| 데이터 아키텍트(Data Architect) | 데이터 모델·아키텍처 설계·관리 |
| 데이터 관리자(DBA) | 데이터베이스 운영·성능 관리 |

## 5. 업무 데이터

**업무 데이터(Business Data)** 는 실제 업무 처리를 통해 생성·활용되는 데이터이다.

### 5.1 업무 데이터 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 마스터 데이터(Master Data) | 업무의 핵심 기준 정보 | 고객, 상품, 직원 |
| 트랜잭션 데이터(Transaction) | 업무 활동으로 생성되는 데이터 | 주문, 결제, 재고 |
| 코드 데이터(Code Data) | 분류·구분을 위한 코드 | 상태코드, 부서코드 |
| 참조 데이터(Reference Data) | 외부 기관의 표준 데이터 | 우편번호, 통화코드 |

### 5.2 마스터 데이터 관리(MDM)

**MDM(Master Data Management)** 은 조직 전체에서 일관된 마스터 데이터를 유지·관리하는 체계.

- **단일 버전의 진실(Single Version of Truth)**: 여러 시스템에서 동일한 마스터 데이터 사용
- 데이터 중복 제거, 데이터 통합 실현

**MDM 구현 방식**:

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 중앙집중형(Centralized) | 단일 MDM 저장소에서 모든 시스템에 배포 | 일관성 최고 | 단일 장애점 |
| 연합형(Federated) | 각 시스템이 자체 마스터 유지, 동기화 | 유연성 높음 | 일관성 관리 어려움 |
| 하이브리드(Hybrid) | 중앙집중+연합 혼합 | 균형적 | 설계 복잡 |

## 출제 포인트

1. **데이터 품질 6차원**: 정확성·완전성·일관성·적시성·유효성·유일성
2. **표준화 구성 4요소**: 표준 단어·표준 용어·표준 코드·표준 도메인
3. **메타데이터 4유형 + 예시**: 기술(테이블구조)·업무(데이터사전)·운영(접근로그)·관리(보안등급)
4. **데이터 오너 vs 데이터 스튜어드**: 오너=최종 책임, 스튜어드=실무 관리
5. **마스터 데이터 vs 트랜잭션 데이터**: 기준정보(고객·상품) vs 업무 활동 데이터(주문·결제)
6. **참조 데이터**: 외부 기관 표준(국가코드·통화코드)
7. **MDM 구현 방식**: 중앙집중형·연합형·하이브리드
8. **단일 버전의 진실(Single Version of Truth)**: MDM의 핵심 목표
9. **완전성**: NULL·공백 비율로 측정
10. **일관성**: 여러 시스템 간 데이터 동일 여부
11. **적시성**: 데이터 생성~반영 지연 시간으로 측정
12. **모델 데이터 품질**: 완전성·일관성·표준 준수·추적성
