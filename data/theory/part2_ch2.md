# 2과목 2장: 정보 요구 사항 조사

## 1. 정보 요구 사항 수집

정보 요구 사항 수집은 이해관계자로부터 업무 요구 정보를 획득하는 활동이다.

### 1.0 정보 요구사항 유형 (4가지)

| 유형 | 설명 | 예시 |
|------|------|------|
| 외부 인터페이스 요건 | 시스템 송수신 간 요구사항 | 타 시스템 연계·API |
| 기능개선 요건 | 시스템 내 프로세스에 대한 요구 | 업무 처리 기능 추가 |
| 성능개선 요건 | 성능 관련 요구 (트랜잭션 시간, 동시 사용자 수 등) | 응답 시간 개선 |
| 보안개선 요건 | 보안 요소 관련 요구 (인증, 암호화, 방화벽 등) | 접근 권한 강화 |

### 1.1 이해관계자(Stakeholder) 식별

정보 요구 조사의 첫 단계는 요구 사항에 영향을 주거나 받는 모든 이해관계자를 식별하는 것이다.

| 유형 | 예시 |
|------|------|
| 내부 이해관계자 | 현업 담당자, IT 부서, 경영진 |
| 외부 이해관계자 | 고객, 파트너사, 규제 기관 |

### 1.2 수집 기법

| 기법 | 특징 | 적합 상황 |
|------|------|---------|
| 인터뷰(Interview) | 1:1 또는 소그룹, 심층 정보 수집 | 핵심 이해관계자 의견 파악 |
| 워크숍(Workshop) | 다수 참여, 합의 도출 | 여러 이해관계자 의견 통합 |
| JAD(Joint Application Development) | 사용자·개발자·관리자 집중 워크숍 | 빠른 요건 도출·합의 |
| 설문(Survey) | 대규모, 정량적 데이터 수집 | 많은 이해관계자 의견 취합 |
| 관찰(Observation) | 현장 업무 직접 관찰 | 실제 업무 방식 파악 |
| 문서 검토 | 기존 문서·시스템 분석 | 현행 업무 현황 파악 |
| 브레인스토밍 | 자유로운 아이디어 제시 | 창의적 요구 발굴 |
| 포커스 그룹 | 소그룹 토론, 의견·태도 수집 | 특정 주제 깊이 있는 탐색 |
| 프로토타입 | 시각적 요건 확인·피드백 | 요건 명확화·조기 검증 |

### 1.2a 면담(Interview) 심화

**면담 역할 분담:**

| 역할 | 책임 |
|------|------|
| **면담자** | 실질적 면담 진행, 대상자 질문 |
| **기록자** | 면담 내용을 있는 그대로 기록하고, 종료 후 재확인 |
| **관찰자** | 주제 이탈 시 주의, 보충 질문, 종료 여부 판단 |

**면담 진행 과정:**
```
대상자 선정 → 일정 수립 → 면담 준비 → 면담 진행 → 결과 분석 → 결과 피드백
```

| 단계 | 주요 내용 |
|------|---------|
| 대상자 선정 | 관련 업무를 명확히 이해하는 사람 (필수), 후원자·사용자 추천 (선택) |
| 일정 수립 | 프로젝트 목적과 범위 전달, 관련 문서 자료 요청 |
| 면담 준비 | 대상자 파악(업무·성향), 시나리오 준비, 30분 전 최종 점검 |
| 면담 진행 | 준비 내용 기반 진행, 필요시 질문 수정 |
| 결과 분석 | 팀 전원 참석, 이슈 정리, 필요시 추가 면담 |
| 결과 피드백 | **반드시** 면담 대상자 본인에게 결과지 승인 |

**면담 구조 유형:**

| 유형 | 특징 | 적합 상황 |
|------|------|---------|
| **피라미드** | 구체적 → 일반적 질문 순 | 아이디어 발굴, 구체적 사항 확인 |
| **퍼널** | 일반적 → 구체적 질문 순 | 기존 아이디어 발전, 범위 좁히기 |
| **다이아몬드** | 피라미드 + 퍼널 결합 | 균형 있는 심층 탐색 |

**면담 시간 기준:**
- 상위 관리자: 1.5시간 이내
- 실무자: 3시간 이내

**워크숍 목적:**
- 경영층·현업 부서장의 공통된 의견 도출
- 유사 업무 부서 간 면담 시간 절감
- 전문가 판단력으로 최적의 결론 도출

### 1.3 수집 시 주요 확인 사항

| 확인 항목 | 내용 |
|---------|------|
| 현재 업무 프로세스 | 현업이 수행하는 실제 업무 흐름 |
| 데이터 항목 | 업무에서 사용하는 데이터 목록 |
| 문제점·개선 요구 | 현재 시스템의 불편사항과 개선 필요 사항 |
| 법규·컴플라이언스 | 반드시 준수해야 하는 법규·정책 |

## 2. 정보 요구 사항 정리

수집된 정보 요구 사항을 구조화·표준화하여 체계적으로 정리하는 활동이다.

### 2.1 정보 요구 사항 분류

수집된 요구 사항을 주제·기능·데이터 영역 등 기준으로 분류한다.

| 분류 기준 | 예시 |
|---------|------|
| 업무 기능별 | 주문 관리, 재고 관리, 고객 관리 |
| 데이터 주제 영역별 | 고객, 상품, 주문, 재고 |
| 요구 유형별 | 기능적/비기능적/데이터 표준/품질 |
| 우선순위별 | 필수/중요/선택 |

### 2.2 정보 요구 사항 정규화(Normalization)

유사하거나 중복된 요구 사항을 통합하고 표준 형식으로 기술하는 작업.

| 정규화 방법 | 내용 |
|---------|------|
| 유사 요구 통합 | 같은 의미의 요구 사항을 하나로 통합 |
| 중복 제거 | 동일 요구 사항 제거 |
| 표준 형식 기술 | 통일된 형식으로 재기술 |
| 모호한 표현 명확화 | 구체적이고 측정 가능한 표현으로 수정 |

### 2.3 정보 요구 사항 명세서 작성

| 항목 | 내용 |
|------|------|
| 요구 ID | 고유 식별자 (예: IR-001) |
| 요구 명칭 | 간결한 명칭 |
| 요구 설명 | 상세한 내용 기술 |
| 출처 | 이해관계자 및 근거 문서 |
| 우선순위 | Must/Should/Could/Won't |
| 관련 업무 | 연관된 업무 기능 |
| 검증 기준 | 충족 여부 확인 방법 |

### 2.4 정보 요구 분류 체계

수집된 정보 요구를 데이터 아키텍처 관점에서 분류:

```
주제 영역(Subject Area)
  └── 핵심 엔터티 후보
        └── 데이터 항목(Attribute)
              └── 업무 규칙(Business Rule)
```

## 3. 정보 요구 사항 통합

여러 이해관계자·부서·시스템에서 수집된 정보 요구 사항을 하나의 일관된 체계로 통합하는 활동이다.

### 3.1 통합의 필요성

| 이유 | 내용 |
|------|------|
| 불일치 해소 | 부서별로 다르게 정의된 요구 사항의 불일치 해결 |
| 중복 제거 | 동일한 요구 사항의 중복 기술 제거 |
| 일관성 확보 | 전사 단일 정보 요구 사항 체계 수립 |
| 누락 보완 | 개별 수집 시 놓친 요구 사항 발굴 |

### 3.2 통합 절차

```
1. 수집된 요구 사항 목록화
2. 동의어·유사어 그룹화
3. 불일치·충돌 식별
4. 이해관계자 협의·조정
5. 통합 요구 사항 확정
6. 전사 정보 요구 사항 명세서 완성
```

### 3.3 이해관계자 간 충돌 조정

| 유형 | 해결 방법 |
|------|---------|
| 용어 불일치 | 표준 용어 선정·용어 사전 등록 |
| 요구 사항 충돌 | 우선순위 기준 또는 조정 회의로 결정 |
| 범위 불일치 | 프로젝트 범위 명확화 |

### 3.4 통합 결과물

| 산출물 | 내용 |
|--------|------|
| 통합 정보 요구 사항 명세서 | 전사 관점의 정보 요구 목록 |
| 데이터 주제 영역 목록 | 정보 분류 체계 |
| 업무 규칙 목록 | 정보 생성·변경·삭제 규칙 |
| 이해관계자 매트릭스 | 요구 사항별 이해관계자 책임 매핑 |

### 3.5 우선순위 결정 방법

**방법 1: 화폐가치 산출**
1. 기업 차원의 중요성 평가 (1~3점)
2. 시스템 차원의 중요성 평가 (1~3점)
3. 다른 요구사항 영향 정도 평가 (1~5점)
4. 전체 점수 합산 후 백분율 환산
5. 백분율을 금액으로 환산하여 우선순위 배정

**방법 2: 상대적 중요도 산정**
1. 업무 기여도 평가 (1~5점)
2. 현행 정보시스템 충족 정도 평가 (1~3점)
3. 요구사항 간 연관성 평가 (최고 9점)
4. 평가 점수 기준으로 가중치 결정

## 출제 포인트

1. JAD: 사용자·개발자·관리자가 함께 참여하는 집중 워크숍 기법
2. 인터뷰 장점: 심층 정보 수집, 단점: 많은 시간 소요
3. 설문 장점: 대규모 정량적 데이터 수집, 단점: 심층 정보 한계
4. 관찰: 현장의 실제 업무 방식 파악에 적합
5. 브레인스토밍: 자유 아이디어 제시, 판단 유보
6. 포커스 그룹: 소그룹 토론, 의견·태도 수집
7. 정보 요구 분류 기준: 업무기능별·주제영역별·요구유형별·우선순위별
8. 정규화: 유사 통합·중복 제거·표준 형식·모호한 표현 명확화
9. 통합 필요성: 불일치 해소·중복 제거·일관성·누락 보완
10. 이해관계자 충돌 해결: 표준 용어 선정, 조정 회의, 범위 명확화
11. 통합 산출물: 통합 정보 요구 명세서·주제영역 목록·업무규칙·이해관계자 매트릭스
