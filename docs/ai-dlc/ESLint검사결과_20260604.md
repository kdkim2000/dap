### LN-001: `react-hooks/exhaustive-deps` — 0건 (완전 통과)

총 18개 `useEffect` 사용 중 ESLint가 경고를 발생시킨 항목 없음.  
대표적 올바른 패턴:

```typescript
// pages/theory/[chapterId].tsx
useEffect(() => { setLastVisited('theory', chapterId) }, [chapterId, setLastVisited])

// pages/practical/[practiceId].tsx
useEffect(() => { draftRef.current = draft }, [draft])

// components/practical/AnswerTextEditor.tsx — 클린업 전용 (의존성 없음 정당)
useEffect(() => () => {
  if (timerRef.current) clearTimeout(timerRef.current)
}, [])
```

### LN-008: `global-require` 억제 — 3건 (불가피한 ESLint 비활성화)

| # | 파일 | 라인 | 이유 |
|:---:|:---|:---:|:---|
| 1 | `lib/questions.ts` | 11 | `require()` 동적 JSON 로드 (Next.js SSG 패턴) |
| 2 | `lib/questions.ts` | 20 | 모의고사 JSON 동적 로드 |
| 3 | `lib/practical.ts` | 6 | 실기 문제 JSON 동적 로드 |

**결론**: Next.js Pages Router SSG에서 `require()` 동적 로딩은 공식 패턴.  
`import()` 비동기 방식으로 대체 가능하나 SSG 맥락에서 `require()`가 더 단순하고 지원됨.  
→ `eslint-disable-next-line` 주석 사용 적절.

## 권장 사항 (개선 기회)

### 권장-1: ESLint 규칙 강화 (선택 사항)

현재 `next/core-web-vitals`만 사용 중. 프로젝트 성숙도를 높이려면 아래 추가 규칙 검토:

```json
{
  "extends": "next/core-web-vitals",
  "rules": {
    "no-console": ["warn", { "allow": ["error", "warn"] }],
    "@typescript-eslint/no-unused-vars": "error"
  }
}
```

→ 프로덕션 코드의 `console.log`를 자동 감지할 수 있게 됨 (낮음 우선순위)

### 권장-2: `global-require` 장기 개선

```typescript
// 현재 (require 방식)
return require(`@/data/questions/${chapterId}.json`) as Question[]

// 개선 방향 (동적 import — SSG 호환성 확인 필요)
const mod = await import(`@/data/questions/${chapterId}.json`)
return mod.default as Question[]
```

→ Next.js 14의 동적 import 지원 범위에서 테스트 필요 (낮음 우선순위)
